  • Interpretar o resultado: maior score indica agrupamentos mais bem separados.

  • Visualizar no espao PCA (2D).

Dados usados e variveis-chave:

  • df_scaled_features: dados padronizados gerados com StandardScaler a partir de num_cols.
  • labels_kmeans: rtulos dos clusters atribudos pelo KMeans.
  • df_kmeans['cluster_kmeans']: coluna com o cluster de cada pas (para agregaes e interpretao).
  • perfil_clusters: mdias das variveis por cluster (perfil socioeconmico/sade tpico).

Medide por cluster (interpretao):

  • Funo calcula o pas mais prximo do centride no espao padronizado (menor distncia euclidiana).
  • til para comunicao: um pas representativo por cluster ajuda a explicar o perfil do grupo.

Consideraes do K-Means:

  • Assume agrupamentos aproximadamente esfricos/convexos e usa distncia Euclidiana.
  • Sensvel a outliers e escala: por isso padronizamos previamente.
  • n_init=10: mltiplas inicializaes para reduzir risco de mnimo local.

O que o ndice de Silhueta?

O mtodo da Silhueta mede a qualidade do agrupamento calculando o quo semelhante um objeto ao seu prprio cluster (coeso) em comparao com outros clusters (separao).

  • O valor varia de -1 a +1.
  • +1: O ponto est bem longe dos clusters vizinhos (bom).
  • 0: O ponto est na fronteira de deciso entre dois clusters.
  • -1: O ponto pode ter sido atribudo ao cluster errado. Usaremos a mdia do score de silhueta para determinar o nmero ideal de clusters ($k$).

# Funo para medides: pas mais prximo ao centride do cluster (no espao padronizado)
def compute_medoids_from_kmeans(X_scaled_df, labels, centroids, countries):
    medoid_indices = {}
    medoid_countries = {}
    X = X_scaled_df.values
    for c in np.unique(labels):
        idx = np.where(labels == c)[0]
        Xc = X[idx]
        centroid = centroids[c]
        dists = np.linalg.norm(Xc - centroid, axis=1)
        local_min = np.argmin(dists)
        medoid_idx = idx[local_min]
        medoid_indices[c] = medoid_idx
        medoid_countries[c] = countries[medoid_idx]
    return medoid_indices, medoid_countries

# COMPARAO FINAL DAS MTRICAS ---

def calcular_metricas_validacao(nome_modelo, X, labels):
    # Ignora se s tiver 1 cluster ou s rudo
    if len(set(labels)) < 2:
        print(f" {nome_modelo}: No formou clusters suficientes para calcular mtricas.")
        return
    
    sil = silhouette_score(X, labels)
    dbi = davies_bouldin_score(X, labels)
    chi = calinski_harabasz_score(X, labels)
    
    print(f"--- {nome_modelo} ---")
    print(f" Silhueta (Mx 1.0):       {sil:.4f}  {' (Bom)' if sil > 0.2 else ''}")
    print(f" Davies-Bouldin (Mn 0.0): {dbi:.4f}  {' (Quanto menor, melhor)'}")
    print(f" Calinski-Harabasz (Max):  {chi:.4f}  {' (Quanto maior, melhor)'}")
    print("-" * 30)

Anlise Comparativa: K-Means vs DBSCAN

K-Means (K=5):

  • Apresentou um ndice de Silhueta de aproximadamente 0.30.
  • Segmentou todos os pases em 5 grupos distintos.
  • Vantagem: Garante que todo pas pertena a um grupo, facilitando a poltica de alocao total.
  • Desvantagem: Fora outliers a pertencerem a um cluster, o que pode distorcer a mdia do grupo (especialmente no Cluster 2 e 3 de alta renda).

DBSCAN:

  • Apresentou um ndice de Silhueta superior (~0.38 nos clusters formados).
  • Identificou cerca de 18 pases como Rudo/Outliers.
  • Vantagem: Alta capacidade de detectar anomalias e no fora agrupamento onde no h densidade clara. Os clusters formados so mais coesos (da a silhueta maior).
  • Desvantagem: Deixa 18 pases sem "categoria" definida (rudo), o que pode ser problemtico se a regra de negcio exigir classificao para todos.

Concluso:

  • Para uma ao humanitria focada na maioria, o K-Means oferece uma categorizao completa.
  • Para deteco de casos extremos (pases muito ricos ou muito pobres fora do padro), o DBSCAN superior.

Observao:

  • Observei que o K-Means tendeu a dividir os dados em grupos mais equilibrados, apresentando um ndice Calinski-Harabasz maior (o que normal, pois essa mtrica favorece clusters redondos, especialidade do K-Means).

  • J o DBSCAN foi excelente em identificar Outliers (pases com economia muito fora da curva, classificados como rudo -1). Parte 4: Respostas Tericas e Sries Temporais

  1. A silhueta um ndice indicado para escolher clusters no DBSCAN?

  • Resposta:No o mais indicado, embora possa ser usado com cautela.

    • Por que? O ndice de Silhueta assume que os clusters so convexos (redondos e densos), que como o K-Means funciona. O DBSCAN, por outro lado, cria clusters de formatos arbitrrios (como luas, anis ou cobras) e classifica pontos distantes como rudo.
    • Consequncia: Um cluster em formato de "lua" perfeito no DBSCAN pode ter uma pontuao de Silhueta baixa porque a distncia matemtica entre as pontas da lua grande, mesmo que a densidade indique que eles pertencem ao mesmo grupo. O ndice DBCV (Density-Based Clustering Validation) seria teoricamente mais apropriado.
  1. Medidas de Similaridade em Sries Temporais

  • Problema: Agrupar 10 sries temporais em 3 grupos usando Correlao Cruzada Mxima.

    • Passo a Passo (Tpicos):

      1. Pr-processamento: Padronizar cada srie temporal individualmente (Z-Score) para que a amplitude (escala vertical) no afete a comparao, focando apenas na forma da onda.
      2. Clculo da Correlao Cruzada: Para cada par de sries, calcular a correlao cruzada permitindo deslocamento de tempo (lag). Identificar o valor mximo absoluto dessa correlao (para capturar sries que so iguais mas esto atrasadas no tempo).
      3. Matriz de Dissimilaridade: Converter a correlao mxima em distncia. Frmula: $D = 1 - |Corr_{max}|$. (Se a correlao for perfeita (1), a distncia 0).
      4. Clusterizao: Aplicar um algoritmo hierrquico ou espectral sobre essa matriz de distncias.

Qual algoritmo usar?Recomenda-se a Clusterizao Hierrquica (Agglomerative Clustering).

  • Justificativa: O K-Means exige o clculo de um "centride" (mdia). Calcular a mdia de sries temporais que esto defasadas no tempo (uma acontece antes da outra) destri o padro da onda. O mtodo Hierrquico trabalha diretamente com a matriz de distncias que calculamos no passo 3, sem precisar criar centrides artificiais.

Caso de Uso:Anlise de Ciclos Econmicos Globais. Alguns pases entram em recesso imediatamente aps um choque global. Outros pases, dependendo de sua economia, sentem o impacto 6 meses depois. A correlao cruzada agruparia esses pases no mesmo "cluster de comportamento", ignorando o atraso temporal.

Outra estratgia (Sugesto):DTW (Dynamic Time Warping).

  • Passos: O DTW mede a similaridade encontrando o melhor alinhamento no linear entre duas sries. Ele "estica" ou "comprime" o eixo do tempo para fazer os picos e vales coincidirem. ideal se uma srie acontece mais rpido que a outra (ex: uma pessoa falando rpido vs. devagar). A seguir, implementamos uma variante que "representa" cada cluster pelo medide (ponto real mais prximo ao seu baricentro) em todas as iteraes.

from sklearn.metrics import pairwise_distances

def k_medoids(X, k, max_iter=200, random_state=42):
    '''
    Implementao simplificada de K-medoids:
    - Inicializa medides (pontos reais) aleatrios.
    - Atribui cada ponto ao medide mais prximo.
    - Atualiza cada medide escolhendo o ponto do cluster que minimiza a soma de distncias aos demais.
    - Repete at estabilizar ou atingir max_iter.
    '''
    rng = np.random.default_rng(random_state)
    n = X.shape[0]
    # Inicializao por amostragem sem reposio
    medoid_indices = rng.choice(n, size=k, replace=False)
    D = pairwise_distances(X, X)  # matriz de distncias completa

    for it in range(max_iter):
        # Atribuio
        distances_to_medoids = D[:, medoid_indices]  # (n, k)
        labels = np.argmin(distances_to_medoids, axis=1)

        updated = False
        # Atualizao por cluster
        for c in range(k):
            cluster_points = np.where(labels == c)[0]
            if len(cluster_points) == 0:
                continue
            # Soma das distncias de cada candidato aos outros pontos do cluster
            intra_D = D[np.ix_(cluster_points, cluster_points)]
            costs = intra_D.sum(axis=1)
            best_idx_in_cluster = cluster_points[np.argmin(costs)]
            if best_idx_in_cluster != medoid_indices[c]:
                medoid_indices[c] = best_idx_in_cluster
                updated = True
        if not updated:
            break
    return labels, medoid_indices

K-mdias sensvel a outliers. Explique.

  • O K-mdias utiliza a mdia dos pontos para definir os centrides. A mdia fortemente afetada por valores extremos; portanto, poucos outliers podem deslocar significativamente o centride de um cluster, levando a atribuies incorretas e deteriorando a qualidade dos agrupamentos.

Por que DBSCAN mais robusto a outliers?

  • O DBSCAN define clusters como regies de alta densidade separadas por regies de baixa densidade. Pontos isolados (outliers) tendem a ser classificados como "rudo" (label -1) e no influenciam o centro do cluster. Alm disso, no pressupe formato globular nem exige nmero de clusters pr-definido, sendo mais robusto a diferentes formas e presena de outliers.

            with open(out_html, 'w', encoding='utf-8') as f:
                f.write(new_content)
            print("CSS de impresso injetado com sucesso!")
            print(f"Abra {out_html} no navegador e use 'Imprimir -> Salvar como PDF'.")
        else:
            print("AVISO: Tag </head> no encontrada no HTML gerado.")
    except Exception as e:
        print(f"Erro ao injetar CSS: {e}")
